## Featured Post Draft

I am building my Cloud/DevOps Support portfolio around a practical problem: small teams often have a VPS running Docker apps, but no clear diagnostics, monitoring, backup process, or restore proof.

My project, CloudOps Rescue Kit, focuses on those basics:
- collect a clean diagnostics report,
- check app endpoints,
- back up Docker Compose volumes,
- test restore steps,
- document the incident and handover.

This is not a SaaS. It is a practical toolkit and case-study system using low-cost tools: Linux, Docker, Bash, cron, Uptime Kuma, and GitHub.

I am using it to demonstrate support-ready skills for Cloud Support, DevOps Support, Linux Support, and Junior DevOps roles.

GitHub: https://github.com/s3nafps/cloudops-rescue-kit
