# Job Search Filters And Daily Workflow

## Target Roles

Primary:
- Cloud Support Engineer
- DevOps Support Engineer
- Technical Support Engineer - Cloud
- Linux Support Engineer
- Application Support Engineer - Linux
- Junior DevOps Engineer

Secondary:
- Platform Support Engineer
- Site Reliability Support Engineer
- Infrastructure Support Engineer
- Customer Engineer - Cloud
- Support Engineer - Docker/Linux

- Algeria
- Remote Algeria

## Filters

Use these defaults:

- Date posted: last 24 hours or last 7 days.
- Experience: entry, associate, junior, mid-level support.
- Remote: remote first, then hybrid only if realistic.
- Job type: full-time first, contract second.
- Language: English; add French when useful.
- Avoid: senior-only Kubernetes/platform roles unless requirements are flexible.

## Daily Application Workflow

Minimum daily target:
- 8 role searches.
- 5 tailored applications.
- 3 direct messages to hiring managers, founders, or support leads.
- 1 GitHub/portfolio improvement or screenshot.
- Update `trackers/daily-application-tracker.csv`.

Quality rule:
- Do not apply with a generic CV if the job mentions Linux, Docker, monitoring, backup, Cloudflare, or customer support. Tailor the summary and top project bullets to match.

## Tailoring Rules

If the job says Docker:
- Lead with CloudOps Rescue Kit, Docker Compose diagnostics, logs, volumes, backup scripts.

If the job says Linux:
- Lead with VPS troubleshooting, logs, disk/memory checks, service status, SSH workflows.

If the job says customer support:
- Lead with clear incident reports, handover notes, before/after evidence, and calm communication.

If the job says monitoring:
- Lead with Uptime Kuma, endpoint checks, alert tests, and first-response runbooks.

If the job says backup/recovery:
- Lead with backup archive, checksum, and restore-test evidence.

## Direct Message Template For Hiring Managers

Hi [Name], I saw your opening for [Role]. I am based in Algeria and focusing on Cloud/DevOps Support: Linux, Docker, VPS deployments, monitoring, backups, and support documentation.

I built a small public project called CloudOps Rescue Kit to show practical support evidence: diagnostics, Docker volume backup, endpoint checks, restore testing, and incident handover notes.

If useful, I can send the GitHub link and a short case study. Thanks.

## Weekly Review

Every Sunday:

- Count applications sent.
- Count replies.
- Count interviews.
- Identify which keywords produced replies.
- Improve CV headline and top project bullet.
- Add one screenshot or case-study improvement.
- Remove job boards that produce no useful leads after 2 weeks.
